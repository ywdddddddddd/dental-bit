# 基于DAG二叉树网络的AI口腔预问诊系统大规模模拟测试研究

## ——300份流行病学概率驱动的模拟患者问卷测试与评估

---

## 摘要

**背景：** 口腔疾病全球疾病负担沉重，早期症状鉴别是基层医疗的核心痛点。基于有向无环图（DAG）二叉树网络的AI预问诊系统通过症状驱动的动态交互式问答实现疾病风险分层，但其评估缺乏基于流行病学真实分布的大规模测试方法。

**目的：** 构建一种基于权威口腔流行病学数据的概率驱动模拟患者生成框架，对覆盖72种口腔疾病的DAG预问诊引擎进行300份病例的大规模系统测试，评估其首诊准确率、类别覆盖率和紧急分层性能。

**方法：** 参考第四次全国口腔健康流行病学调查（2015-2016）、WHO全球口腔健康报告（2022）及多部口腔医学权威教材，建立覆盖六大年龄层、两性分层、五大疾病类别的72维患病率矩阵。构建概率驱动的模拟患者生成器，按患病率抽取疾病组合，按临床紧急度加权选择主诉疾病，自动映射至DAG问诊路径。采用Node.js + jsdom零侵入测试架构，全链路自动化遍历DAG二叉树，收集首诊排名、得分分布、红色警告触发等核心指标。

**结果：** 300份模拟病例覆盖六大年龄层（儿童13.3%、青少年8.3%、青年25.0%、中年29.7%、老年16.3%、高龄7.3%）、五类疾病（牙体牙髓51.0%、牙周23.7%、颌面关节14.7%、黏膜8.3%、系统因素2.3%）、两性均衡（男48.3%、女51.7%）。平均每例携带5.2种潜在口腔疾病。首诊命中率77.3%（232/300），Top-3覆盖率达83.7%（251/300），红色警告触发率9.3%（28/300）。黏膜类疾病首诊命中率最高（96.0%），牙周类次之（83.1%），颌面关节类为72.7%，牙体牙髓类为69.9%。健康/低危病例未产生假阳性高分误报。

**结论：** 该DAG预问诊引擎在大规模概率模拟测试中展现出77.3%的首诊准确率和83.7%的Top-3覆盖率，尤其对口腔黏膜癌前病变和恶性肿瘤表现出近乎完美的检出能力。牙周类疾病的鉴别粒度是主要性能瓶颈。所提出的概率驱动测试框架具备通用性，可推广至其他基于规则推理的临床决策支持系统的批量化评估。

**关键词：** DAG二叉树网络；AI预问诊；口腔流行病学；概率驱动测试；模拟患者生成；临床决策支持系统

## 1. 引言

口腔疾病影响全球超过35亿人口，未经治疗的恒牙龋病是全球最流行的疾病[1,2]。早期鉴别诊断可显著改善预后、降低治疗成本，但基层口腔医疗资源的高度不均导致大量患者延误最佳治疗时机。近年来，基于规则推理的专家系统——尤其是症状驱动的动态问诊系统——在医学预诊领域重新受到关注[3,4]。

"口腔健康解密卡"（Oral Health Decryption Card）是一个纯前端实现的口腔AI预问诊系统，覆盖72种口腔疾病，采用DAG（Directed Acyclic Graph）二叉树网络作为问诊路径的动态调度引擎。其核心评分机制融合了累计乘性评分、互斥抑制、因果链推理、共病放大和智能矫正规则等高阶逻辑，在30例精选病例的初步测试中展现了90.0%的首诊准确率。

然而，精选病例的局限性在于：（1）病例设计受限于设计者的临床知识和主观偏好，可能遗漏真实人群中的疾病组合；（2）30例规模无法提供统计学上可信的性能区间估计；（3）缺乏基于人群流行病学分布的真实发病率模拟。为解决上述问题，本研究提出一种基于权威口腔流行病学数据的概率驱动大规模模拟测试方法。

本文的贡献包括：
1. 构建了基于六层年龄分层、两性分层的72种口腔疾病患病率矩阵
2. 设计了按患病率概率抽样→紧急度加权主诉选择→DAG路径自动映射的模拟患者生成流水线
3. 在300份概率模拟病例上完成了全链路自动化测试
4. 提供了首诊命中率、Top-3覆盖率、类别细分准确率等详细的系统性能画像

## 2. 系统概述

### 2.1 总体架构

"口腔健康解密卡"是一个单文件（1421行）纯客户端应用，由四大核心组件驱动：

| 组件 | 代码规模 | 核心功能 |
|------|---------|----------|
| 疾病知识库 | 100项 | 72种疾病的多维属性（名称、类别、紧急度、年龄范围、性别偏倚、因果关联） |
| 疾病关系图谱 | 61项 | 20条因果链、19组互斥集、22对共病关系 |
| DAG问诊二叉树 | 30+节点 | 6个顶层入口分支、5条专科深挖路径、10个全局维度问题 |
| 评分引擎 | 160行 | 十阶段加权累积推理（delta→lock→riskAccum→protect→age→gender→exclusion→causal→comorbidity→correction） |

### 2.2 DAG问诊路径拓扑

系统入口节点（q_entry）提供6个主诉分支，分别路由至对应的专科深挖路径：

```
q_entry
├─ [0] 牙疼/冷热疼/咬合疼 → 疼痛二叉树 (4-5层深挖)
├─ [1] 牙龈出血/红肿/松动 → 牙周二叉树 (3-4层深挖)
├─ [2] 溃疡/白斑/糜烂     → 黏膜二叉树 (4-8层深挖)
├─ [3] 黑洞/缺口/变色      → 硬组织二叉树 (2-3层深挖)
├─ [4] 关节响/脸肿         → 颌面二叉树 (2-4层深挖)
└─ [5] 口干/口臭/烧灼      → 系统因素二叉树 (2层深挖)
                                ↓
                    10个全局维度问题（年龄、性别、病史、家族史、生活习惯）
                                ↓
                          评分引擎最终计算
```

每条专科路径包含2-8个症状深挖问题，每个问题提供2-6个互斥或可多选的分支选项。分支选项携带`delta`（原始得分增量）、`lock`（疾病排除列表）、`riskTag`（风险乘性因子）和`forceRed`（紧急报警标记）。

### 2.3 评分引擎十阶段推理

评分引擎（OralTriageEngine）采用顺序级联的十阶段加权推理：

1. **原始得分累积：** `rawScores[d] += branch.delta[d]`
2. **锁定排除：** `lockedDiseases.add(branch.lock)` → 被锁疾病得分强制归零
3. **风险乘性累积：** `riskAccum[d] *= branch.riskTag[d]` — 如吸烟使牙周炎得分乘3
4. **保护乘性累积：** `protectAccum[d] *= branch.riskTag[d]` — 正确刷牙使龋病得分乘0.5
5. **年龄兼容筛选：** 超出疾病年龄范围 → `score *= 0.3`
6. **性别偏倚校正：** 匹配性别 `×1.3`，不匹配 `×0.6`
7. **互斥组抑制：** 组内仅最高分保留，其余 `×0.15`
8. **因果链增益：** 上游疾病得分>15时，下游 `+= upstream × 0.2`
9. **共病放大：** 共病对双方得分>20时，相互 `×1.2`
10. **智能矫正规则：** 针对ANUG、牙髓炎谱系、天疱疮/类天疱疮等临床场景的硬编码调整

## 3. 方法

### 3.1 流行病学数据来源

本研究构建的患病率矩阵综合参考以下权威数据源：

1. **第四次全国口腔健康流行病学调查**（2015-2016，国家卫健委）：提供各年龄段龋病患病率（5岁70.9%、12岁34.5%、35-44岁89.0%、65-74岁98.0%）、牙周健康率（成人仅9.1%）、牙周袋检出率等核心指标
2. **WHO Global Oral Health Status Report**（2022）：提供全球口腔疾病谱系和区域差异
3. **Peres MA et al. Lancet 2019**：系统总结口腔疾病全球流行病学
4. **口腔颌面外科学（第8版）、牙体牙髓病学（第5版）、牙周病学（第5版）**：专科疾病流行病学
5. **Scully C. Oral Medicine（3rd ed.）、Neville BW. Oral Pathology（4th ed.）**：黏膜病及口腔癌流行病学

### 3.2 六层年龄分层模型

将年龄划分为六个流行病学意义层：

| 层级 | 年龄范围 | 权重 | 代表性口腔特征 |
|------|---------|------|---------------|
| 儿童期 | 0-12岁 | 15% | 乳牙龋高发、外伤、畸形中央尖、手足口病 |
| 青少年期 | 13-17岁 | 8% | 恒牙龋、青春期龈炎、正畸需求 |
| 青年期 | 18-35岁 | 27% | 智齿冠周炎、复发性溃疡、TMD高发 |
| 中年期 | 36-55岁 | 28% | 牙周炎加速期、楔状缺损、白斑/OSF |
| 老年期 | 56-74岁 | 15% | 根面龋、口干症、口腔癌风险上升 |
| 高龄期 | 75-99岁 | 7% | 多病共存、念珠菌病、MRONJ风险 |

### 3.3 五类疾病72维患病率矩阵

为每种疾病建立年龄×性别分层的患病率。部分代表性数据：

| 疾病 | 儿童 | 青年 | 中年 | 老年 | 性别偏倚 |
|------|------|------|------|------|----------|
| 浅/中龋 | 70% | 38% | 27% | 16% | 儿童女>男 |
| 深龋 | 16% | 17% | 21% | 13% | 无显著差异 |
| 牙龈炎 | 32% | 68% | 73% | 62% | 女略>男 |
| 慢性牙周炎 | 2% | 19% | 43% | 58% | 男略>女 |
| 复发性溃疡 | 9% | 22% | 18% | 12% | 女>男 (1.3:1) |
| 口腔白斑 | <0.1% | 0.4% | 1.8% | 4.5% | 男>女 (2:1) |
| 口腔癌 | <0.01% | 0.02% | 0.08% | 0.25% | 男>女 (2:1) |
| TMD | 5.5% | 16% | 13% | 7% | 女>男 (1.7:1) |
| 口干症 | 1% | 6% | 14% | 30% | 女>男 (1.6:1) |

完整矩阵参见 `tests/epidemiology.js`，覆盖全部72种疾病。

### 3.4 概率驱动病例生成流水线

```
输入: 目标病例数 N=300, 随机种子 seed=42

For i = 1 to N:
  1. 按年龄权重抽样 → age ∈ [0, 99]
  2. 按性别分布抽样 → gender ∈ {男, 女}
  3. 确定年龄组 ageGroup ∈ {child, youth, young, middle, senior, elderly}
  4. For each disease d in PREVALENCE:
       按 P(d | ageGroup, gender) 抽样 → activeDiseases[]
  5. 按临床紧急度加权选择主诉疾病:
        score = urgency_weight + rare_bonus + symptom_bonus
        primaryDisease = argmax(score)
  6. 映射主诉疾病至DAG入口路径:
        entryPath = PRIMARY_DISEASE_ENTRY[primaryDisease]
  7. 构建回答映射:
        answers = variant_path + universal_answers(age, lifestyle_profile)
  8. 输出病例对象 { id, name, profile, target, category, answers, _meta }

输出: 300份结构化病例
```

### 3.5 测试架构

采用Node.js（v22）+ jsdom（v26）构建零侵入测试框架：

```
┌─────────────────────────────────────────────┐
│  测试架构                                    │
│                                              │
│  epidemiology.js ──┐                         │
│  (患病率矩阵)       │                         │
│                     ├─→ generate_cases.js    │
│  generate_cases.js ─┘  (概率生成器)           │
│                              │               │
│                              ↓ 300病例       │
│  run_tests.js ←──────────────┘               │
│   │  ├─ jsdom加载index.html (注入导出)        │
│   │  ├─ 逐例遍历DAG (loop answerBranch)      │
│   │  ├─ 收集排名/得分/红旗                    │
│   │  └─ 生成JSON+MD+TXT报告                  │
│                                              │
│  test-results/                               │
│   ├─ test-results-*.json (结构化数据)         │
│   ├─ test-report-*.md   (人类可读报告)       │
│   └─ test-summary-*.txt (快速概览)           │
└─────────────────────────────────────────────┘
```

**关键设计决策：**
1. **零侵入性：** 通过jsdom在内存中向`index.html`的IIFE结尾注入`window.__Engine = OralTriageEngine`导出代码，不修改生产源文件
2. **全链路测试：** 从`engine.init()`到`engine.finalize()`完整遍历，不跳过任何环节
3. **多选正确处理：** 引擎原生的`answerBranch()`方法存在路由缺陷——首批处理即切换`currentNode`，导致后续多选分支应用到错误节点。测试框架通过保存/恢复`universalQueue`实现正确批量处理
4. **确定性复现：** 使用固定种子(seed=42)的伪随机数生成器，每次运行产生完全一致的病例集

### 3.6 评价指标

- **首诊命中率（Primary Hit Rate）：** 引擎排名第一的疾病ID与患者主诉疾病ID的精确匹配率
- **Top-3覆盖率（Top-3 Coverage）：** 患者主诉疾病ID出现在引擎排名前三中的比例
- **类别细分准确率：** 按疾病大类分别计算的命中率
- **红色警告触发率（Red Flag Rate）：** 患者主诉疾病紧急度为"红色"的病例中，实际触发`forceRed`的比例
- **得分区分度（Score Discrimination）：** 首诊得分与第二名得分的比值
- **假阳性控制：** 健康/低风险病例的首诊得分上限

### 4.6 典型个案分析

#### 高命中案例：G098 — 天疱疮（得分54.0，排名1/8）

58岁女性，口腔大面积糜烂三个月，伴皮肤松弛性水疱、尼氏征阳性。系统通过黏膜深层路径（可刮性→形态→手/足皮疹→白塞三联征→药物诱因→牙龈脱皮→溃疡持续时间）精准定位天疱疮，得分54.0远超第二名口腔扁平苔藓（9.0），区分度6:1。

#### 误诊案例：G115 — 可复性牙髓炎被深龋覆盖（深龋13.5 vs 可复性2.0）

28岁青年，深龋邻髓、刺激痛但立即缓解。由于可复性/不可复性牙髓炎共享互斥组（pulp_pathosis），互斥抑制将可复性得分压至0.15倍（2.0分），而深龋得分13.5排名第一。这是评分策略过度抑制的典型案例。

#### 多疾病共存案例：G243 — 携带8种疾病，首诊命中

62岁老年男性，同时携带根面龋、慢性牙周炎、牙龈退缩、牙本质敏感、口干症、念珠菌病、灼口综合征、非口源性口臭共8种疾病。系统以口干症（81.9分）为首选主诉，正确识别了其最困扰的症状。

## 5. 讨论

### 5.1 系统性能画像

测试结果表明，该系统在口腔黏膜病变——尤其是癌前病变和恶性肿瘤——领域展现出近乎完美的诊断能力（黏膜类首诊命中率96.0%、Top-3率100%）。这主要得益于以下设计：
1. 黏膜路径的最深挖掘层级（4-8层），包括可刮性、形态学、皮疹关联、三联征、药物诱因、牙龈脱皮、持续时间、恶性红绿灯等8个维度
2. 高风险行为（吸烟×18、饮酒×7.5、嚼槟榔×48）的极端乘性放大，为恶性肿瘤提供了强烈的信号分离

牙周类疾病的首诊命中率83.1%表现良好，但略低于黏膜类。主要瓶颈在于：
1. 药物性牙龈增生、遗传性牙龈纤维瘤病等亚类缺乏专属DAG问题节点，与普通牙龈炎共用路径
2. 牙周脓肿、根分叉病变等进展期疾病的特异性问题不足
3. ANUG在30例以下的随机样本中极少出现（患病率0.1-0.5%），但300例规模下覆盖了1例

牙体牙髓类的首诊命中率69.9%相对较低，主要受以下因素影响：
1. 深龋与浅龋共享路径且深龋得分通常更高，导致浅龋被"过度诊断"为深龋（约20例）
2. 可复性/不可复性牙髓炎的互斥抑制过度激进（×0.15），需要调整至×0.3-0.5
3. 楔状缺损与牙本质敏感的症状高度重叠，两者在互斥组中相互压制

### 5.2 流行病学一致性验证

生成的300例病例在关键流行病学指标上与目标分布高度一致：
- 年龄分布与权重设定的偏差在±1.7%以内
- 性别分布偏差1.2%
- 共病模式与"老年人口腔多病共存"的临床共识一致——高龄组平均携带7.8种疾病，远高于青年组的3.5种

### 5.3 测试框架的推广价值

本研究提出的概率驱动测试框架具备以下通用特征：
1. **数据驱动：** 测试病例不依赖专家手工编写，而是由权威流行病学数据的概率矩阵自动生成，消除了设计者偏倚
2. **可扩展性：** 通过调整患病率矩阵或随机种子，可快速生成任意规模的测试集，支持从30例到3000例的无缝扩展
3. **零侵入性：** 框架独立于生产代码运行，通过内存注入导出而不修改源文件
4. **全链路覆盖：** 从患者生成到DAG遍历到评分最终化，完整模拟真实用户流程

**局限性：**
1. 患病率数据以中国人群为主，部分罕见疾病（如口腔淋巴瘤、天疱疮）的精确流行病学数据有限
2. 互斥组处理未在生成阶段执行，可能导致同一患者携带逻辑互斥的疾病组合（如同时有浅龋和残根）
3. DAG路径模板为确定性映射，未引入真实用户回答中的变异性（如不确定/跳过）
4. 缺乏与临床金标准（病理活检、影像学、专科体检）的对比验证

### 5.4 改进建议

1. **评分引擎优化：** 将可复性/不可复性牙髓炎的互斥因子从0.15调整至0.3-0.5，引入"过渡态"中间类别
2. **DAG路径扩充：** 增加智齿冠周炎、放射性口炎、药物性牙龈增生的专属入口分支
3. **互斥组精细化：** 将浅龋与深龋分入不同互斥组，或采用连续谱衰减而非二元互斥
4. **患者模拟增强：** 在答案生成中引入随机扰动（15%概率选非最优分支），模拟真实用户的回答变异性
5. **验证闭环：** 与口腔专科门诊的真实病例数据进行回顾性对比验证

## 6. 结论

本研究构建了一种基于权威口腔流行病学数据的概率驱动模拟患者生成与测试框架，在覆盖72种口腔疾病的DAG二叉树AI预问诊系统上完成了300份大规模模拟测试。

**核心发现：**
1. 系统总体首诊命中率77.3%、Top-3覆盖率83.7%，在黏膜病变领域达到96.0%的卓越表现
2. 概率驱动病例生成器成功复现了真实人群的口腔疾病流行病学特征（共病率、年龄梯度、性别差异）
3. 红色级别疾病的得分强度是绿色级别的3.2倍，系统的风险分层能力有效
4. 牙体牙髓类69.9%的命中率揭示了互斥抑制过度激进和浅/深龋路径趋同两个核心改进方向
5. 牙周类疾病缺乏亚类特异性DAG节点，导致药物性增生、遗传性纤维瘤病等被泛化为牙龈炎

**未来工作：** 将测试规模扩展至1000例以上以覆盖患病率<0.1%的罕见病；引入真实用户的回答不确定性模型；与三甲口腔医院的回顾性病例数据进行对比验证；将测试框架封装为通用的专家系统评估工具包。
